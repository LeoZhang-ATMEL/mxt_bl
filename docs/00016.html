Running The Application
  1. Connect a micro USB cable to the DEBUG port.
  2. Build and program the application using the MPLAB X IDE.
  3. Press and hold the User Switch as mentioned in below table and then press Reset button or Power cycle the device to force trigger bootloader at startup. The User LED will be turned on to indicate that the bootloader code is running.
    • Note: The User LED will also turn on when the bootloader does not find a valid application; i.e. the first word of the application (stack pointer), contains 0xFFFFFFFF.

  1. The below steps are needed only when the bootloader itself is being updated. To update the bootloader, the fuse settings must be disabled and binary must be generated. Launch MHC for the I2C bootloader application

 

  • Disable Fuse Settings

 

  • Specifing post build option to generate binary file:

The below option must be specified in Project Properties > Building > Execute this line after build, to automatically generate the binary file from hex file once the build is complete 

${MP_CC_DIR}\xc32-objcopy -I ihex -O binary "${DISTDIR}/${PROJECTNAME}.${IMAGE_TYPE}.hex" "${DISTDIR}/${PROJECTNAME}.${IMAGE_TYPE}.bin"

 

  • Build the project
  • Refer to Running The Application section of either the bootloader_host_nvm (Step 2 onwards) or bootloader_host_sdcard (Step 3 onwards) for steps required to upgrade the bootloader.
